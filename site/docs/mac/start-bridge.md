Bridge は、iPhone と Mac 上の Codex をつなぐ待ち受けです。`CodexPocketMac` では `Bridge を有効にする` がオンなら、起動時に自動で始まります。

## まず見る場所

- `一般` の `ランタイム`
- `一般` の `ヘルス`
- `ペアリング` の QR
- `Bridge` のローカル endpoint

`ランタイム` が `実行中` か `外部で実行中` なら、たいていそのまま Pairing に進めます。

## 手で始める方法

自動で始まっていないときは、次のどちらかを使います。

- `一般` タブで `Bridge を有効にする` をオンにする
- メニューバーから `Bridge を開始` を選ぶ

## 正常に始まったかを確認する

次の状態になっていれば準備完了です。

- `ランタイム` が `実行中`
- `ヘルス` が表示される
- `ペアリング` タブで QR を表示できる
- `Bridge` タブにローカル endpoint が表示される

## 通常は触らなくてよい項目

通常は初期値のままで構いませんが、必要なら `Bridge` タブで次を変えられます。

- 表示名
- ポート
- Pairing 用メモ

変更後は `保存` を押して状態を更新してください。

## 席を離れて使うなら

Mac の前にいない時間が長いなら、`一般` の `ログイン時に起動` だけは見ておくと安心です。Bridge 自体は起動後に自動で立ち上がる設計なので、毎回ここで手動操作する前提ではありません。

## うまく始まらないとき

- `詳細` タブで `Codex CLI` と `シェル` の状態を確認する
- `ログ` タブで stdout / stderr を確認する
- 古い launchd 設定が残っているときは `一般` や `詳細` の警告を確認する
