description: stable と next の意味、そして公開の切り替わり方を説明します。
---

このドキュメントサイトには、公開済みの内容と、次の版の内容があります。CodexPocket は iPhone app の審査と Mac 配布をそろえて出すため、`next` にある内容がすぐ public になるとは限りません。使っている版に合わせて読み分けると、画面名や手順の違いで迷いにくくなります。

## まず見分けたい 2 つ

| 版 | URL | どういうときに読むか |
| --- | --- | --- |
| stable | `/docs/` | 公開済みの iPhone / Mac / docs をそのまま使っているとき |
| next | `/docs/next/` | 未公開の変更、審査中の版、案内された検証版を見ているとき |

現在の stable は `0.1.4` です。

## 迷ったら stable を見る

通常は `stable` を見れば十分です。配布されているアプリをそのまま使っているなら、まず `stable` の説明が合います。

`next` は、次の版に向けた更新内容です。手元のアプリより先の説明が含まれることがあるため、特に理由がない限りは `stable` を優先してください。

## 公開はどう切り替わるか

CodexPocket の公開は、だいたい次の順で進みます。

1. iPhone build を App Store Connect に upload して審査へ出す
2. そのあいだ docs の current は `next` として更新されることがある
3. iPhone 審査が通ったあとで、Mac 配布、リリースノート、docs stable が同じ版へ切り替わる

このため、`next` には「これから出る版」や「審査中の版」の説明が入ることがあります。まだ公開されていないなら、通常の利用者は `stable` を見てください。

## release notes と版番号の見方

Mac 側の公開配布物は GitHub Releases で管理しています。release notes を開くときは、通常の download ページか、今回案内された版番号を確認してください。

- download / release notes:
  `codex-pocket-releases` の GitHub Releases
- docs の stable:
  公開済みの版
- docs の next:
  次の公開候補

画面に表示されている版番号、Mac の release notes、docs の version 表示がそろっていれば、その説明をそのまま使えます。

## ドキュメントと画面が少し違うとき

開いているページの版と、手元のアプリの版がずれている可能性があります。画面右上のバージョン切り替えで `stable` と `next` を確認してください。

また、開発版、検証版、審査用 build を使っている場合は、ドキュメントより先に画面が変わっていることがあります。その場合は、実際のアプリに表示されているラベルと案内された release notes を優先して問題ありません。

## この章で覚えておけば十分なこと

- 迷ったら `stable`
- 試験的な案内、審査中の版、未公開版を見るときだけ `next`
- iPhone 審査が終わるまでは、`next` の内容がそのまま public とは限らない
- 画面名が合わないときは、まずバージョン表示を確認する
