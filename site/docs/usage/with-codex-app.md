---
title: Codex App と一緒に使う
description: Mac の Codex App を使っている人向けの連携方法をまとめます。
---

Mac で Codex App を使っているなら、iPhone 側で Project を毎回手動追加する必要はありません。Codex App に登録・利用している workspace が、CodexPocket 側の Project 一覧に入ってくることがあります。

Mac でいつも開いている workspace を、そのまま iPhone の入口にしたい人にはこの使い方がいちばん自然です。

## おすすめの流れ

1. Mac の Codex App で、普段使う workspace を開くか登録しておく
2. `CodexPocketMac` を起動し、`Bridge` が動いていることを確認する
3. Mac の `プロジェクト` タブに同じ workspace が出ているか確認する
4. iPhone の `ホーム` または `プロジェクト` を下に引いて更新する

これで、Mac でふだん触っている workspace を iPhone 側でもそのまま開きやすくなります。

## 自動でそろいやすいもの

- workspace の場所
- Project の表示名
- Codex App 側で使っている workspace の並びや増減

細かな同期の仕組みを意識する必要はありません。ユーザーとしては「Mac の Codex App にある workspace が、CodexPocketMac の Project 一覧にも見えてくる」と覚えておけば十分です。

## 手動で追加したほうが早い場面

- 使いたい workspace が一覧に出てこない
- 表示名を自分で分かりやすく付け直したい
- いまこの場で iPhone 側からすぐ Project を増やしたい

その場合は、iPhone 側の `プロジェクト` で `+` を押して追加します。

## 使い始めるときのコツ

- Project が見えたら、まずはそのまま開いて `Thread` 一覧を確認します。
- 見たい Thread がまだないときは、iPhone から `新しい Thread` を始めるのが最短です。
- Mac 側の作業環境が本体なので、iPhone は「同じ workspace を離れた場所から開く窓口」と考えると迷いません。

Mac の Codex App と CodexPocketMac を一緒に使うと、普段の workspace 管理は Mac のまま、再開と追従だけを iPhone に任せられます。
