# iPhone Setup

This page is the starting scaffold for the shortest path into CodexPocket from the iPhone app. We will add screenshots and detailed steps later, but the information layout is fixed around this flow.

## What must already be ready on Mac

- the macOS companion app is running
- the bridge is started and can show a pairing QR or manual code
- for the first release, the iPhone and Mac are on the same LAN

## First-run flow on iPhone

1. Add a Host through QR import or manual entry
2. wait for bridge verification and Project catalog collection to finish
3. open the new Host and choose the target Project
4. confirm that Threads, branches, skills, and commands are available

## Main screens after pairing

- `Projects`
  switches the workspace context for threads, exec, and branches
- `Threads`
  resumes existing threads and opens transcript history
- `Composer`
  sends follow-up turns, slash commands, skills, and exec
- `Settings`
  adjusts app language, theme, font size, and follow-up behavior
