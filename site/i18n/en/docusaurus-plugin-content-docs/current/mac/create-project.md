title: Create a Project
description: This page explains how to make a working folder show up as a CodexPocket project on the iPhone.
---

In CodexPocket, the unit shown on the iPhone is called a `Project`. Workspaces you already use in Codex App on the Mac can become `Project` candidates as they are.

![Diagram of workspaces flowing from Codex App to CodexPocketMac and the iPhone](/img/docs/workspace-sync.svg)

## The easiest path

1. Put the folder you use every day into the Codex App workspace list on the Mac.
2. Open `Projects` in `CodexPocketMac`.
3. Confirm that the same folder is visible as a `Project`.

The `Project` name may be filled automatically. If you want a clearer display name later, adjust it from add or edit on the iPhone.

## One project is enough at first

If just one `Project` is visible before pairing, the iPhone can import the list right away. It is fine to start with only the workspaces you use most and add the rest later.

## What kinds of folders fit well

- repositories where you keep continuing threads
- work folders whose progress you want to check while away from the desk
- workspaces you open often for research or review

## If it does not appear automatically

- wait a few seconds while keeping `CodexPocketMac` open
- reopen the `Projects` tab
- if it still does not appear, use [Register a Project](./register-project)
