---
title: Install the Codex App
description: This page explains what to confirm before linking CodexPocketMac for people who already use Codex App.
---

If you already use Codex App on the Mac, this page is more of a checklist than an installation guide. `CodexPocketMac` reads workspace information from Codex App and uses it as the source for `Project` candidates shown on the iPhone.

## What to confirm here

- Codex App works normally on the Mac
- the folders you want to see on the iPhone are already in the Codex App workspace list
- those same folders appear in `Projects` inside `CodexPocketMac` after a short wait

## Things worth remembering

- you do not need to keep Codex App open all the time
- `CodexPocketMac` imports the workspace list automatically
- use manual add from the iPhone only when the list does not appear

## How to confirm the reflection

1. In Codex App, confirm that the folder you want is visible as a workspace.
2. Open `CodexPocketMac`.
3. Check whether the same folder appears in the `Projects` tab.

If it does not appear immediately after you add it, wait a few seconds and check again.

## If you do not use Codex App

CodexPocket still works when your workflow is based on the `codex` CLI alone. In that case, follow [Register a Project](./register-project) and add the working directory from the iPhone.
