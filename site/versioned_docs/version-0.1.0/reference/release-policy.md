---
title: リリースと更新の見方
description: このドキュメントで stable と next をどう読み分けるかを説明します。
---

このドキュメントサイトには、公開済みの内容と、次の版の内容があります。使っているアプリに合わせて読み分けると、画面名や手順の違いで迷いにくくなります。

## まず見分けたい 2 つ

| 版 | URL | どういうときに読むか |
| --- | --- | --- |
| stable | `/docs/` | 公開済みアプリを使っているとき |
| next | `/docs/next/` | 未公開の変更を試しているとき、または案内されたとき |

現在の stable は `0.1.2` です。

## 迷ったら stable を見る

通常は `stable` を見れば十分です。配布されているアプリをそのまま使っているなら、まず `stable` の説明が合います。

`next` は、次の版に向けた更新内容です。手元のアプリより先の説明が含まれることがあるため、特に理由がない限りは `stable` を優先してください。

## ドキュメントと画面が少し違うとき

開いているページの版と、手元のアプリの版がずれている可能性があります。画面右上のバージョン切り替えで `stable` と `next` を確認してください。

また、開発版や検証版を使っている場合は、ドキュメントより先に画面が変わっていることがあります。その場合は、実際のアプリに表示されているラベルを優先して問題ありません。

## この章で覚えておけば十分なこと

- 迷ったら `stable`
- 試験的な案内を受けたときだけ `next`
- 画面名が合わないときは、まずバージョン表示を確認する
