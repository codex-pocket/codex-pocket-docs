---
title: 接続を確認する
description: ペアリング後に連携が正しく完了したか確認する方法をまとめます。
---

Pairing が終わったら、次の順番で確認すると迷いません。Mac に慣れている人ほど、「Bridge は動いているか」「iPhone に Project が見えているか」の 2 点を先に見ると判断が早くなります。

## 最短で確認するならここを見る

1. Mac の `一般` で `ランタイム` が `実行中` になっているか確認します。
2. iPhone の `ホスト` で追加した Host が `初回設定中` から `接続確認済み` に変わるか確認します。
3. iPhone の `プロジェクト` に Mac 側の `Project` が表示されるか確認します。
4. そのまま `Project` を開き、`Thread` 一覧まで進めるか確認します。

## iPhone 側で見る場所

- `ホーム`
  既定 Host の状態と、最近使った `Project`
- `ホスト`
  Host ごとの状態、`接続テスト`、`設定を再試行`
- `プロジェクト`
  取り込まれた `Project` 一覧

`ホスト` 画面では、Host を左にスワイプすると `接続テスト` を実行できます。保存直後は `初回設定中`、完了後は `接続確認済み` と表示されるのが目安です。

## Mac 側で見る場所

- `一般`
  `ランタイム` と `ヘルス`
- `Bridge`
  現在の local endpoint
- `ペアリング`
  QR、現在の endpoint、トークン
- `ログ`
  接続に失敗したときの確認先

## 正常時の目安

- Mac 側の `ランタイム` が `実行中`
- Mac 側の `ヘルス` が `ok`
- iPhone 側の Host が `接続確認済み` になる
- `プロジェクト` に 1 件以上の `Project` が表示される
- `Project` を開くと `Thread` 一覧または `新しい Thread` の導線まで進める

## 少し待てば戻ることがある状態

- Host がまだ `初回設定中`
- `接続テスト` が `接続中`
- `Thread` 画面のライブ更新が一時的に止まる
- iPhone をバックグラウンドにしたあと、表示の再同期に少し時間がかかる

この場合でも、Mac 側では Bridge や Codex の処理が続いていることがあります。まずは iPhone を前面に戻して待つか、`接続テスト`、画面の引き下げ更新、`設定を再試行` を試してください。
