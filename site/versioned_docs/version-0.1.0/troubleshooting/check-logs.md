---
title: ログを確認する
description: 接続や実行で詰まったときに Mac 側のログを見る方法を説明します。
---

詳しい原因を追いたいときは、Mac 側のログを確認します。iPhone 側ではなく、Mac アプリにログが集まる前提で考えると分かりやすくなります。

## まず見る場所

CodexPocketMac の `ログ` タブを開きます。

- `stdout`
- `stderr`

ここに、Bridge 起動失敗や実行時のエラーが出ます。

## あわせて見る場所

- `一般`
  直近のエラー表示
- `詳細`
  Codex CLI、git、シェルの検出状態
- `ログフォルダを開く`
  外部に共有したいとき

## どんなときにログを見るか

- Bridge が開始できない
- Pairing 後の初回設定が進まない
- Project 一覧が取れない
- Thread が失敗する

## 見つけやすいエラーの例

- Codex CLI が見つからない
- git が見つからない
- ポート競合
- ヘルスチェック失敗

原因が分からないときは、`一般` と `詳細` の表示を見てから `ログ` に入ると、読みやすくなります。
