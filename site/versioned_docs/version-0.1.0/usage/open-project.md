title: Project を開く

Project は、iPhone から開く workspace の単位です。Mac 上のどのフォルダで Codex を動かすかを、iPhone では Project として選びます。

同じリポジトリでも Mac が違えば別の Project です。まずは「どの Mac の、どの workspace を開くか」を選ぶ場所だと考えると分かりやすくなります。

## Project を開く場所

- `ホーム`
  最近使った Project やお気に入りから、すぐ再開したいときに向いています。
- `プロジェクト`
  すべての Project を一覧で見たいときに向いています。

## 開く手順

1. `ホーム` または `プロジェクト` を開く
2. 対象の Project をタップする
3. ブランチと Thread の情報が読み込まれるのを待つ
4. 既存 Thread を開くか、`新しい Thread` を始める

Project を切り替えると、その Project 用の Thread 一覧、ブランチ情報、コマンドやスキルの候補も一緒に切り替わります。

## `ホーム` と `プロジェクト` の使い分け

- よく使う Project は `ホーム` の `最近使ったプロジェクト` や `お気に入り` から開く
- たまに使う Project や、一覧を見比べたいときは `プロジェクト` を開く
- 複数の Mac を登録している場合は、同じ名前の Project でも Host を見て選ぶ

## Project 一覧でできること

- お気に入りに追加
- 順番を入れ替える
- 編集
- 削除
- 新しい Project の追加

`プロジェクト` タブの `+` から追加するときは、Host を選んだうえで `作業ディレクトリ` を指定します。`Browse on Mac` を使うと、Mac 側のフォルダを見ながら選べます。

## 迷ったとき

- まずは普段いちばんよく使う workspace を 1 件だけお気に入りにしておく
- 一覧が古そうなら、画面を下に引いて更新する
- Project が見つからないときは [Codex App と一緒に使う](./with-codex-app) か [Project を追加する](../mac/register-project) を確認する

最初は Project を増やしすぎないほうが、外出先での切り替えが楽です。毎日使うものから順にそろえると、`ホーム` だけでも十分回せるようになります。
